## Pre-built binaries

lychee provides binaries for Linux, macOS, and Windows for every release.
You can download them from the [releases page](https://github.com/lycheeverse/lychee/releases).

## Docker

lychee provides an official Docker image at [lycheeverse/lychee](https://hub.docker.com/r/lycheeverse/lychee),
which is updated on every release.

To check files on your local machine, mount them into the Docker container:

<Tabs>
  <TabItem label="Linux/macOS">
    <Code
      code="docker run --init -it -v $(pwd):/input lycheeverse/lychee /input/README.md"
      lang="bash"
    />
  </TabItem>
  <TabItem label="Windows (PowerShell)">
    <Code
      code="docker run --init -it -v ${PWD}:/input lycheeverse/lychee /input/README.md"
      lang="powershell"
    />
  </TabItem>
  <TabItem label="Windows (CMD)">
    <Code
      code="docker run --init -it -v %cd%:/input lycheeverse/lychee /input/README.md"
      lang="cmd"
    />
  </TabItem>
</Tabs>

The `--init` parameter allows lychee to be stopped from the terminal.
The `-it` flags start an interactive terminal, which is required to show the progress bar.

## Building from source

Install Rust if you haven't already:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

On Linux, install build dependencies:

<Tabs>
  <TabItem label="Debian/Ubuntu">
    <Code
      code="sudo apt install gcc pkg-config libc6-dev libssl-dev"
      lang="bash"
    />
  </TabItem>
  <TabItem label="Fedora/RHEL">
    <Code code="sudo dnf install gcc pkg-config openssl-devel" lang="bash" />
  </TabItem>
  <TabItem label="Arch">
    <Code code="sudo pacman -S base-devel openssl" lang="bash" />
  </TabItem>
</Tabs>

Then install lychee:

```bash
cargo install lychee
```

### Feature flags

lychee supports several optional features:

- `native-tls` - Platform-native TLS via [native-tls](https://crates.io/crates/native-tls) (default)
- `vendored-openssl` - Statically link OpenSSL
- `rustls-tls` - Pure Rust TLS via [rustls](https://crates.io/crates/rustls)
- `email-check` - Validate email addresses (default)
- `check_example_domains` - Allow checking example domains

Install with specific features:

```bash
cargo install lychee --no-default-features --features rustls-tls
cargo install lychee --features vendored-openssl
```

## Usage

### Check all links in the current directory

```bash
lychee .
```

### Check a website

```bash
lychee https://example.com
```

:::note
This checks links on the given page only. For entire sites, download the sitemap and pass it as input.
:::

### Check specific files

```bash
lychee README.md
lychee test.html info.txt
lychee test.html info.txt https://example.com
```

### Check links offline

```bash
lychee --offline path/to/directory
```

### Check a remote file

```bash
lychee https://raw.githubusercontent.com/lycheeverse/lychee/master/README.md
```

### Check from stdin

```bash
cat test.md | lychee -
echo 'https://example.com' | lychee -
```

### Use glob patterns

```bash
lychee 'docs/**/*.md'
```

:::note[Glob Expansion Rules]
Use single quotes to prevent shell expansion.
:::

### Advanced globbing

```bash
lychee '~/projects/*/README.md'
lychee --glob-ignore-case '~/projects/*/[r]eadme.*'
```

### Check EPUB files

If you have [atool](https://www.nongnu.org/atool) installed:

```bash
acat -F zip file.epub "*.xhtml" "*.html" | lychee -
```

:::caution
lychee parses other file formats as plaintext and extracts links using [linkify](https://github.com/robinst/linkify).
This works well for most cases, but if you need dedicated support for a specific format, [create an issue](https://github.com/lycheeverse/lychee/issues).
:::

## GitHub Action

lychee is available as a [GitHub Action](https://github.com/lycheeverse/lychee-action/):

```yaml
- name: Link Checker
  uses: lycheeverse/lychee-action@v2
  with:
    args: --verbose --no-progress './**/*.md' './**/*.html'
```

See our [GitHub Actions guide](/guides/github-actions) for more examples.
