---
title: Local File Checking with --root-dir
description: Use --root-dir to specify where lychee should look for files referenced by absolute paths.
---

The `--root-dir` parameter tells lychee where to look for files referenced by absolute paths (links starting with `/`).

## When to Use

Use `--root-dir` when your HTML contains absolute paths:

```html title="index.html"
<a href="/about.html">About</a>
<a href="/docs/guide.html">Guide</a>
```

If your site is built in a `public` directory:

```bash
lychee --root-dir "$(pwd)/public" "**/*.html"
```

lychee will look for:
- `./public/about.html`
- `./public/docs/guide.html`

:::note
`--root-dir` must be an absolute path.
:::

## Examples

### Static Site Builder

```bash
my-site/
├── public/           # Generated files
│   ├── about.html
│   └── docs/
│       └── guide.html
```

Check the links:

```bash
lychee --root-dir "$(pwd)/public" "public/**/*.html"
```

### GitHub Actions

```yaml
- uses: lycheeverse/lychee-action@v2
  with:
    args: >-
      --root-dir "${{ github.workspace }}/public"
      "public/**/*.html"
```

### Documentation Site

```bash
lychee --root-dir "$(pwd)" "docs/**/*.html"
```

## `--root-dir` vs `--base-url`

- `--root-dir` - Filesystem path for finding local files (links starting with `/`)
- `--base-url` - URL for resolving relative links (like `./guide.html`)

### Using Both

```bash
lychee \
  --root-dir "$(pwd)/public" \
  --base-url https://example.com/ \
  "public/**/*.html"
```

## Configuration File

```toml title="lychee.toml"
root_dir = "/absolute/path/to/public"
```
