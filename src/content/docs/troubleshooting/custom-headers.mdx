---
title: Custom Headers
description: Configure custom HTTP headers for sites that require specific headers to return valid responses.
---

Some sites require custom headers to return valid responses. For example, crates.io expects an `Accept: text/html` header or returns a [404 error](https://github.com/rust-lang/crates.io/issues/788).

## Adding Custom Headers

Pass headers using the `--header` flag:

```bash
lychee --header "accept=text/html" https://crates.io
```

## Multiple Headers

Use the flag multiple times for multiple headers:

```bash
lychee --header "accept=text/html" --header "user-agent=custom" .
```

## Accept All Content Types

Accept any content/MIME type:

```bash
lychee --header "accept=*/*" .
```

## Configuration File

```toml title="lychee.toml"
headers = [
    "accept=text/html",
    "user-agent=custom"
]
```

:::tip
Learn more about the `Accept` header at [MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Accept).
:::
